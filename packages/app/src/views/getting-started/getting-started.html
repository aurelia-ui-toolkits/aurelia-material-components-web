Getting started

For help getting started with a new Aurelia app, check out the Aurelia CLI.

For existing apps, follow this guide to use Aurelia MDC.

Install Aurelia MDC

Install all packages at once

npm

npm i @aurelia-mdc-web/all@au2

yarn

yarn add @aurelia-mdc-web/all@au2 or individual packages you wish to use, e.g.

npm

npm i @aurelia-mdc-web/button@au2

yarn

yarn add @aurelia-mdc-web/button@au2

Roboto font

To get started, first include the Roboto font with the 300, 400 and 500 weights. You can host it yourself or include it from Google Fonts: ${robotoLink}

Material Icons

If you want to use Material Design Icons, load the icon font in your index.html: ${materialIconsLink} For more information on using Material Icons, check out the Material Icons Guide.

SVGs

Some components, such as the circular-progress require the SVGAnalyzer from Aurelia to be included and regsiterd in your Aurelia 2 Application. To do so add the following to your main.ts/js file. import { SVGAnalyzer } from '@aurelia/runtime-html'; .register(SVGAnalyzer);

Aurelia Webpack Loader

It is important to setup a Webpack rule to load custom elements templates using the Aurelia Webpack Loader. Add the following to your webpack.config.js: module: { rules: [ { test: /@aurelia-mdc-web.*\.html$/i, use: '@aurelia/webpack-loader' }, ] },

MDC Sass modules - Webpack

In your webpack.config.js, make sure node_modules/ is listed as a Sass include path. This is needed for the Sass compiler to be able to find the MDC Web Sass files. Explicitly setting sass implementation is also a good idea because sass-loader picks node-sass if there is one. Even if it is installed globally! This is bound to fail as node-sass does not support new @use syntax. ... loader: 'sass-loader', options: { implementation: require('sass'), sassOptions: { includePaths: [path.resolve('./node_modules')] } } ...

MDC Sass modules - Aurelia CLI bundler

Be sure to use Sass as your css pre-processer.

In your aurelia_project\tasks\process-css.ts, add includePaths: ["node_modules"] to the Sass options. This is needed for the Sass compiler to be able to find the MDC Web Sass files.
This example uses gulp-dart-sass. import { build } from 'aurelia-cli'; import * as gulp from 'gulp'; import * as project from '../aurelia.json'; import * as sass from 'gulp-dart-sass'; export default function processCSS() { return gulp.src(project.cssProcessor.source, { sourcemaps: true }) .pipe(sass.sync({ includePaths: ["node_modules"] }).on('error', sass.logError)) .pipe(build.bundle()); } // styles.scss @use './styles/body'; @use '@material/theme' with ( $primary: #6200ee, $secondary: #faab1a, $background: #fff, ); // all styles @use "@aurelia-mdc-web/all"; // or individual packages @use "@material/button/mdc-button"; // MDC Typography @use '@material/typography/mdc-typography'; // MDC Button @use './styles/button'; // styles/_body.scss // Override user agent body margin for mdc-top-app-bar body { margin: 0; } // styles/_button.scss @use '@material/button/mdc-button'; @use '@material/button'; @use '@material/theme'; .button-primary { @include button.filled-accessible(theme.$primary); } .button-secondary { @include button.filled-accessible(theme.$secondary); }

Configure Aurelia MDC plugins

Configure each plugin you want to use. // main.ts import { AllConfiguration } from '@aurelia-mdc-web/all'; import { ButtonConfiguration } from '@aurelia-mdc-web/button'; Aurelia // all packages .register(AllConfiguration) // or individual .register(ButtonConfiguration) .app(Root) .start();
